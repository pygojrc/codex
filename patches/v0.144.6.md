# Codex 0.144.6 Termux patch set

## Fixed upstream base

- Tag: `rust-v0.144.6`
- Commit: `5d1fbf26c43abc65a203928b2e31561cb039e06d`
- Previous audited Termux base: `rust-v0.144.4`
- Published target: `termux-v0.144.6`
- Build target: `aarch64-linux-android`

## Upstream delta adopted

The stable update changes these upstream files:

- `codex-rs/core/src/exec_policy.rs`
- `codex-rs/core/src/exec_policy_tests.rs`
- `codex-rs/core/tests/suite/exec_policy.rs`
- `codex-rs/shell-command/src/bash.rs`
- `codex-rs/shell-command/src/command_safety/is_dangerous_command.rs`
- `codex-rs/models-manager/models.json`
- `codex-rs/Cargo.toml`

The first six files use the official `rust-v0.144.6` versions. They provide the expanded dangerous-command checks and refreshed GPT-5.6 Sol, Terra, and Luna metadata, including `272000`-token context windows.

For `codex-rs/Cargo.toml`, the upstream workspace version is adopted while the existing reviewed Termux release profile and Android-specific configuration remain present.

## Termux compatibility delta retained

The following reviewed compatibility and supply-chain controls remain required after the upstream merge:

1. Browser authentication uses `termux-open-url` on Android.
2. The Android binary targets `aarch64-linux-android` and resolves sibling `libc++_shared.so` through `$ORIGIN` RUNPATH.
3. Android-specific OpenSSL, PTY, clipboard, filesystem-lock and TLS-root compatibility paths remain target-gated.
4. Runtime update surfaces point only to `pygojrc/codex` releases and do not execute a third-party npm updater.
5. The app-server daemon resolves its own executable with `current_exe()`, keeps automatic updates disabled and creates owner-only control sockets.
6. Android `rusty_v8` archives and bindings are selected by version and SHA-256-pinned in `third_party/v8/android-artifacts.toml`.
7. The release workflow builds only Android ARM64, uses `cargo build --locked`, separates read-only build permissions from release write permissions and publishes a checksum with the archive.

## Required verification

Before merging to `main`:

```bash
CODEX_PUBLIC_SANITIZED_TREE=1 bash verify-patches.sh
cd codex-rs
cargo fmt --all -- --check
cargo metadata --locked --no-deps --format-version 1 >/dev/null
```

The release workflow must produce only:

- `codex-termux-aarch64-0.144.6.tar.gz`
- `codex-termux-aarch64-0.144.6.tar.gz.sha256`

No x86, x86_64, desktop Linux or macOS package is part of this Termux release.
