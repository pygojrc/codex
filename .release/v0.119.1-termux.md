## Termux fork notes

This hotfix keeps the public fork on upstream `rust-v0.119.0` and republishes the
latest Android Termux line as `0.119.1-termux`.

Hotfix included in this release:
- Fixed Termux session bootstrap after the upstream `installation_id` file-locking
  change by tolerating unsupported file locks on Android.

Fork-specific changes still carried in this line:
- Android browser login still uses `termux-open-url`.
- Self-update and release links point to `DioNanos/codex-termux`.
- npm update/install flows target `@mmmbuto/codex-cli-termux@latest`.
- Voice and realtime audio remain disabled for the Termux package.
- Launcher hardening preserves helper re-exec behavior on Android.
- Android ELF `RUNPATH=$ORIGIN` hardening from fork PR #1 is included so bundled
  `libc++_shared.so` resolves on direct native invocation.

Install or update:

```bash
npm install -g @mmmbuto/codex-cli-termux@latest
```

Known limitation: the packaged Termux build does not ship voice input or realtime
audio support.

## Upstream 0.119.0 highlights

- Realtime voice sessions moved forward significantly upstream, but the Termux
  package continues to ship without voice/realtime audio support.
- MCP Apps and custom MCP server support improved, including better resource
  reads, tool-call metadata, uploads, and cache refresh behavior.
- Remote/app-server workflows gained better websocket transport, remote `--cd`,
  runtime remote-control enablement, sandbox-aware filesystem APIs, and an
  experimental `codex exec-server` command.
- The TUI adds `Ctrl+O` to copy the latest agent response and improves resume
  and notification behavior.
- Startup, `/status`, `/resume`, MCP inventory, sandbox edge cases, and several
  platform-specific issues were fixed upstream.
