# v0.133.0

This release updates the Android Termux fork to upstream OpenAI Codex
`rust-v0.133.0` and publishes it as `@mmmbuto/codex-cli-termux@0.133.0`.

## Termux package

- Android ARM64 native package for Termux.
- `codex` and `codex-exec` packaged together.
- Browser login remains routed through `termux-open-url`.
- Fork updates stay on `DioNanos/codex-termux` and
  `@mmmbuto/codex-cli-termux`.
- Managed update guidance stays on the fork-owned npm package.
- App-server daemon standalone update fetching remains disabled for the Termux
  fork, with `autoUpdateEnabled` kept false by default.
- Packaged launchers preserve `CODEX_SELF_EXE` and bundled
  `libc++_shared.so` visibility.
- Android ELFs use `RUNPATH=$ORIGIN` for direct native invocation.
- Voice and realtime audio remain disabled in the published Termux package.
- Maintainer builds use fork-owned Android `rusty_v8` artifacts (updated to
  v147.4.0).

## Upstream base

- OpenAI Codex `rust-v0.133.0`

## Upstream highlights

- Goals are now enabled by default, backed by dedicated storage, and track
  progress across active turns.
- `codex remote-control` now runs like a foreground command, waits for
  readiness, reports machine status, and keeps explicit daemon-style
  `start`/`stop` commands.
- Permission profiles gained list APIs, inheritance, managed
  `requirements.toml` support, runtime refresh behavior, and stronger Windows
  sandbox integration.
- Plugin discovery is easier to inspect, with marketplace-aware list output,
  installed versions, visible marketplace roots, and remote collection support.
- Extensions can observe more lifecycle events, including subagent start/stop,
  tool execution, turn metadata, and async approval/turn processing.

Full upstream notes: https://github.com/openai/codex/releases/tag/rust-v0.133.0
