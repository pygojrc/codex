# Termux fork notes

This release rebases the public Termux fork on upstream `rust-v0.125.0` and republishes the Android Termux line as `0.125.0-termux`.

Fork-specific changes kept in this release:
- Android browser login still uses `termux-open-url`.
- Self-update and release links point to `DioNanos/codex-termux`.
- npm update/install flows target `@mmmbuto/codex-cli-termux@latest`.
- Voice and realtime audio remain disabled for the published Termux package.
- `exec`/code-mode remains disabled in the published Android package.
- Launcher hardening preserves helper re-exec behavior on Android.
- Android ELF `RUNPATH=$ORIGIN` hardening is included so bundled `libc++_shared.so` resolves on direct native invocation.
- Unsupported installation-id and arg0 file locks are tolerated on Android so session bootstrap works on Termux.
- Android PTY compatibility is carried by the local `openpty` shim.

Install or update:

```bash
npm install -g @mmmbuto/codex-cli-termux@latest
```

Known limitations:
- the packaged Termux build does not ship voice input or realtime audio support
- the packaged Termux build does not expose `exec`/code-mode

Validation:
- Runtime suite passed on Android (Termux) for `@mmmbuto/codex-cli-termux@0.125.0-termux`.
- Report: `test-reports/latest/0.125.0-termux/CODEX_TEST_REPORT_v0.125.0-termux_run_20260426-2258.md`

## Upstream 0.125.0 highlights

- App-server added Unix socket transport, sticky environments, paginated resume/fork, and remote thread config/store plumbing.
- Plugin management can install remote plugins and auto-upgrade configured marketplaces via the app-server.
- Permission profiles now survive TUI sessions, user turns, MCP sandbox state, and app-server APIs.
- Model providers can own model discovery; AWS account state is exposed to app clients.
- `codex exec --json` reports reasoning-token usage.
- Rollout tracing records tool, code-mode, session, and multi-agent relationships.
