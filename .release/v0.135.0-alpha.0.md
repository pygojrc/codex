# v0.135.0-alpha.0

This pre-release updates the Android Termux fork to upstream OpenAI Codex
`rust-v0.135.0-alpha.2` and publishes it as
`@mmmbuto/codex-cli-termux@0.135.0-alpha.0` on npm `next`.

## Termux package

- Android ARM64 native package for Termux.
- `codex` and `codex-exec` packaged together.
- Browser login remains routed through `termux-open-url`.
- Fork updates stay on `DioNanos/codex-termux` and
  `@mmmbuto/codex-cli-termux`.
- Managed update guidance stays on the fork-owned npm package.
- App-server daemon standalone update fetching remains disabled for the Termux
  fork, with `autoUpdateEnabled` kept false by default.
- `codex remote-control` continues to start/stop cleanly on Termux storage
  backends that reject `flock(2)` with `ENOTSUP` / `EOPNOTSUPP` (Patch #17 +
  the 0.134.1 ENOTSUP file lock guards across `app-server-transport`,
  `core/installation_id`, `arg0`, `message-history`, `execpolicy`).
- Packaged launchers preserve `CODEX_SELF_EXE` and bundled
  `libc++_shared.so` visibility.
- Android ELFs use `RUNPATH=$ORIGIN` for direct native invocation.
- Voice and realtime audio remain disabled in the published Termux package.
- Maintainer builds reuse the fork-owned Android `rusty_v8` prebuilt manifest
  (V8 pin verified against upstream `Cargo.lock`).
- `npx`-spawned MCP servers continue to inherit the Termux env allowlist
  (`PREFIX`, `LD_PRELOAD`, `LD_LIBRARY_PATH`, `NPM_CONFIG_PREFIX`,
  `ANDROID_*`, `XDG_*`) when running on Termux, via runtime
  `TERMUX_VERSION` detection (0.134.1 fix carried forward).

## Upstream base

- OpenAI Codex `rust-v0.135.0-alpha.2`

## Upstream highlights

- New TUI features: markdown table rendering with key-value records and
  app-style layout, OSC 8 web links in rich content, configurable turn
  interruption keybind, vim text object bindings.
- Goal extension: usage limits handled in goal extension; goal accounting
  restored after thread resume.
- App-server: passive probe of running daemon's app-server version surfaced
  in `codex doctor`; turns page included on thread resume.
- MCP setup improvements continue: per-server env targeting, OAuth options
  for streamable HTTP servers, parallel read-only tool dispatch.
- TUI fixes: zellij raw-output handling, tmux modifyOtherKeys fallback,
  macOS stderr composer isolation, markdown list and table improvements.

Full upstream notes: https://github.com/openai/codex/releases/tag/rust-v0.135.0-alpha.2

## Patch inventory

20 patch checks + Bazel inventory PASS on this release. Patch #18 check
target updated to the renamed helper `is_unsupported_file_lock_error`
(kind-based runtime detection; the previous cfg-gated
`installation_id_lock_is_optional` was retired in 0.134.1 because the Termux
release line builds for `aarch64-unknown-linux-musl` where
`cfg!(target_os = "android")` is FALSE).
