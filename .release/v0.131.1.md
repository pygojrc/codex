# v0.131.1

This release keeps the Android Termux fork on upstream OpenAI Codex
`rust-v0.131.0` and publishes it as `@mmmbuto/codex-cli-termux@0.131.1`.

## Termux fork changes

- Android ARM64 native package for Termux.
- `codex` and `codex-exec` packaged together.
- Browser login remains routed through `termux-open-url`.
- Fork updates stay on `DioNanos/codex-termux` and
  `@mmmbuto/codex-cli-termux`.
- Managed update guidance stays on the fork-owned npm package.
- App-server daemon standalone update fetching remains disabled for the Termux
  fork, with `autoUpdateEnabled` kept false by default.
- Packaged launchers preserve `CODEX_SELF_EXE` and bundled
  `libc++_shared.so` visibility.
- Android ELFs use `RUNPATH=$ORIGIN` for direct native invocation.
- Voice and realtime audio remain disabled in the published Termux package.
- Code-mode Android stubs remain aligned with the upstream 0.131.0 API.
- Maintainer builds use fork-owned Android `rusty_v8` artifacts.

## Upstream base

- OpenAI Codex `rust-v0.131.0`

## Upstream highlights

- Richer TUI session controls and display metadata, including service-tier
  commands, blended token usage, permission/approval display, effective
  workspace roots, and responsive Markdown tables.
- Unified `@` mentions for files, directories, plugins, and skills, backed by
  app-server plugin metadata.
- Plugin marketplace and sharing workflow improvements, including version-aware
  sharing, clearer shared-workspace buckets, and default-enabled plugin hooks.
- Daemon-managed remote-control APIs, runtime enable/disable controls, status
  reads, and configured remote environments.
- Python SDK updates under the `openai-codex` / `openai_codex` line, with
  runtime-generated types and approval-mode coverage.
- `codex doctor` diagnostics for runtime, auth, terminal, network, config, and
  local state.
