## Termux fork notes

This release rebases the public fork on upstream `rust-v0.120.0` and republishes
the latest Android Termux line as `0.120.0-termux`.

Fork-specific changes kept in this release:
- Android browser login still uses `termux-open-url`.
- Self-update and release links point to `DioNanos/codex-termux`.
- npm update/install flows target `@mmmbuto/codex-cli-termux@latest`.
- Voice and realtime audio remain disabled for the Termux package.
- Launcher hardening preserves helper re-exec behavior on Android.
- Android ELF `RUNPATH=$ORIGIN` hardening from fork PR #1 is included so bundled
  `libc++_shared.so` resolves on direct native invocation.
- Unsupported installation-id file locks are tolerated on Android so session
  bootstrap works on Termux.

Install or update:

```bash
npm install -g @mmmbuto/codex-cli-termux@latest
```

Known limitation: the packaged Termux build does not ship voice input or realtime
audio support.

## Upstream 0.120.0 highlights

- Realtime V2 can stream background agent progress while work is still running.
- Hook activity in the TUI is easier to scan, with active and completed hook
  output separated more clearly.
- Custom TUI status lines can include the renamed thread title.
- Code-mode tool declarations now expose MCP `outputSchema` details.
- Remote websocket client TLS bootstrap was fixed upstream.
- Sandbox handling for symlinked writable roots was fixed upstream.
