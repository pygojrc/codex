# v0.135.0

Stable release of the Android Termux fork, published as
`@mmmbuto/codex-cli-termux@0.135.0` on npm `latest`.

This is a version-only promotion from the validated `0.135.0-alpha.0` lane:
upstream `rust-v0.135.0` differs from `rust-v0.135.0-alpha.2` solely by the
release version bump, and the published binaries are the same ones already
built in CI and smoke-tested on the alpha `next` lane.

## Upstream base

- OpenAI Codex `rust-v0.135.0`
- Merge path: `rust-v0.134.0` → `rust-v0.135.0-alpha.2` (86 commits) → `rust-v0.135.0` (version bump only)

## Termux fork changes (carry-forward, intact)

- Android ARM64 native package for Termux; `codex` and `codex-exec` packaged together.
- Browser login routed through `termux-open-url`.
- Fork self-update points to `DioNanos/codex-termux` and `@mmmbuto/codex-cli-termux`.
- Packaged launchers preserve `CODEX_SELF_EXE` and bundled `libc++_shared.so` visibility.
- Android ELFs use `RUNPATH=$ORIGIN` for direct native invocation.
- Voice and realtime audio remain disabled in the published Termux package.
- `flock` ENOTSUP/EOPNOTSUPP tolerance across the 7 callsite guards.
- `TERMUX_ENV_VARS` allowlist for `npx`-spawned MCP servers.
- Maintainer builds use fork-owned Android `rusty_v8` artifacts.

## Install

```bash
npm install -g @mmmbuto/codex-cli-termux@latest
```

## Verification

- Termux patch inventory: `bash verify-patches.sh` — all patches present.
- Android ARM64 release build: GitHub Actions `termux-npm-build-publish` (build-only; npm publish is local).
- Pre-publish review: merge APPROVE (version-only, carry-forward patches intact); publish gated on release-hygiene completion.

## Publish checklist (local, from maintainer host)

1. `npm publish <tarball> --tag latest` (single package, Android arm64 payload).
2. `npm view @mmmbuto/codex-cli-termux dist-tags --json` → confirm `latest=0.135.0`.
3. Smoke: `npm install -g @mmmbuto/codex-cli-termux@latest` on Termux arm64, `codex --version`.

## dist-tag / deprecation note

- `latest` → `0.135.0`.
- The alpha `next` (`0.135.0-alpha.0`) is superseded by this stable; deprecate it after stable validation per the publish runbook, or leave it documented as a historical prerelease.
