# v0.130.0

This release merges upstream OpenAI Codex `rust-v0.130.0` into the Android
Termux fork and publishes it as `@mmmbuto/codex-cli-termux@0.130.0`.

## Upstream base

- OpenAI Codex `rust-v0.130.0`

## Upstream highlights

- Plugin details show bundled hooks, and plugin sharing includes link metadata
  and discoverability controls.
- `codex remote-control` provides a simpler headless app-server entrypoint.
- App-server clients can page large thread histories.
- Bedrock auth can use AWS console-login credentials from `aws login` profiles.
- `view_image` can resolve files through the selected environment.

## Termux fork changes

- Android ARM64 native package for Termux.
- `codex` and `codex-exec` packaged together.
- Browser login remains routed through `termux-open-url`.
- Fork self-update points to `DioNanos/codex-termux` and
  `@mmmbuto/codex-cli-termux`.
- Packaged launchers preserve `CODEX_SELF_EXE` and bundled
  `libc++_shared.so` visibility.
- Android ELFs use `RUNPATH=$ORIGIN` for direct native invocation.
- Voice and realtime audio remain disabled in the published Termux package.
- Maintainer builds use fork-owned Android `rusty_v8` artifacts.
