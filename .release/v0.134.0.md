# v0.134.0

This release updates the Android Termux fork to upstream OpenAI Codex
`rust-v0.134.0` and publishes it as `@mmmbuto/codex-cli-termux@0.134.0`.

## Termux package

- Android ARM64 native package for Termux.
- `codex` and `codex-exec` packaged together.
- Browser login remains routed through `termux-open-url`.
- Fork updates stay on `DioNanos/codex-termux` and
  `@mmmbuto/codex-cli-termux`.
- Managed update guidance stays on the fork-owned npm package.
- App-server daemon standalone update fetching remains disabled for the Termux
  fork, with `autoUpdateEnabled` kept false by default.
- `codex remote-control` continues to start/stop cleanly on Termux storage
  backends that reject `flock(2)` with `ENOTSUP` / `EOPNOTSUPP` (Patch #17,
  carried from 0.133.1).
- Packaged launchers preserve `CODEX_SELF_EXE` and bundled
  `libc++_shared.so` visibility.
- Android ELFs use `RUNPATH=$ORIGIN` for direct native invocation.
- Voice and realtime audio remain disabled in the published Termux package.
- Maintainer builds reuse the fork-owned Android `rusty_v8` v147.4.0
  prebuilt (V8 pin unchanged between upstream 0.133.0 and 0.134.0).

## Upstream base

- OpenAI Codex `rust-v0.134.0` (stable)

## Upstream highlights

- Added search across local conversation history, including
  case-insensitive content matches with result previews.
- Made `--profile` the primary profile selector across CLI, TUI permissions,
  and sandbox flows, with legacy profile configs rejected through migration
  guidance.
- Improved MCP setup with per-server environment targeting and OAuth options
  for streamable HTTP servers.
- Made connector tool schemas more reliable by preserving local `$ref`/`$defs`
  structures and compacting oversized schemas before exposure.
- Let read-only MCP tools run concurrently when they advertise `readOnlyHint`.
- Added richer extension and hook context, including conversation history for
  extension tools and subagent identity in hook inputs.
- Goal accounting restored after thread resume.
- Memory state moved to a dedicated SQLite DB.
- TUI: keep raw output above composer in zellij; keep inaccessible apps out
  of mentions.

Full upstream notes: https://github.com/openai/codex/releases/tag/rust-v0.134.0

  installation_id flock tolerance, openpty C symbol for Bionic)

- Patch #20 - Android code-mode stubs
- Patch #21 - Android cross-build vendored OpenSSL
- Patch #22 - V8 Android prebuilt infrastructure

No new code patches were added in this release; the inventory now reflects
the existing fork delta accurately.
