# v0.132.0

This release updates the Android Termux fork to upstream OpenAI Codex
`rust-v0.132.0` and publishes it as `@mmmbuto/codex-cli-termux@0.132.0`.

## Termux package

- Android ARM64 native package for Termux.
- `codex` and `codex-exec` packaged together.
- Browser login remains routed through `termux-open-url`.
- Fork updates stay on `DioNanos/codex-termux` and
  `@mmmbuto/codex-cli-termux`.
- Managed update guidance stays on the fork-owned npm package.
- App-server daemon standalone update fetching remains disabled for the Termux
  fork, with `autoUpdateEnabled` kept false by default.
- Packaged launchers preserve `CODEX_SELF_EXE` and bundled
  `libc++_shared.so` visibility.
- Android ELFs use `RUNPATH=$ORIGIN` for direct native invocation.
- Voice and realtime audio remain disabled in the published Termux package.
- Maintainer builds use fork-owned Android `rusty_v8` artifacts.

## Upstream base

- OpenAI Codex `rust-v0.132.0`

## Upstream highlights

- Python SDK authentication now includes API key login, ChatGPT browser and
  device-code flows, account inspection, and logout APIs.
- Python turn APIs accept plain string input for text-only workflows and return
  richer `TurnResult` metadata for handle-based runs.
- `codex exec resume` supports `--output-schema` for structured resumed
  automations.
- TUI startup probes are batched for faster first-frame rendering.
- Remote executor registration can use standard Codex auth.
- App-server turns preserve requested image fidelity, including
  original-resolution local images.

Full upstream notes: https://github.com/openai/codex/releases/tag/rust-v0.132.0
