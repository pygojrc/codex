# Termux fork notes

This release rebases the public Termux fork on upstream `rust-v0.124.0` and republishes the Android Termux line as `0.124.0-termux`.

Fork-specific changes kept in this release:
- Android browser login still uses `termux-open-url`.
- Self-update and release links point to `DioNanos/codex-termux`.
- npm update/install flows target `@mmmbuto/codex-cli-termux@latest`.
- Voice and realtime audio remain disabled for the published Termux package.
- `exec`/code-mode remains disabled in the published Android package.
- Launcher hardening preserves helper re-exec behavior on Android.
- Android ELF `RUNPATH=$ORIGIN` hardening is included so bundled `libc++_shared.so` resolves on direct native invocation.
- Unsupported installation-id and arg0 file locks are tolerated on Android so session bootstrap works on Termux.
- Android PTY compatibility is carried by the local `openpty` shim.

Install or update:

```bash
npm install -g @mmmbuto/codex-cli-termux@latest
```

Known limitations:
- the packaged Termux build does not ship voice input or realtime audio support
- the packaged Termux build does not expose `exec`/code-mode

## Upstream 0.124.0 highlights

- Reasoning controls in the TUI can now be adjusted quickly from the keyboard.
- App-server sessions gained stronger multi-environment and per-turn environment selection support.
- OpenAI-compatible providers now include first-class Amazon Bedrock support with SigV4 signing.
- Remote plugin marketplaces can be listed and read directly with better details and larger result sets.
- Hooks are now stable and can be configured inline or via managed requirements.
- Eligible ChatGPT plans now default to the Fast service tier unless explicitly overridden.
